Generate retrieval queries for a multilingual knowledge base (Korean and
English). The goal is maximum recall on the user's information need without
diluting precision.

User query:
{user_query}

Context:
{context}

# Goal

Produce exactly 6 search queries that, taken together, cover the user's
information need across both languages while preserving important
identifiers, names, numbers, dates, technical terms, and acronyms.

# Required Queries

Detect the user query language. The "same language" below means the user's
language; the "other language" is the opposite of Korean ↔ English.

1. **Semantic, same language** — natural rephrasing of the full intent.
2. **Semantic, other language** — translate the same intent. Translate
   identifiers/acronyms only if a recognized translation exists; otherwise
   keep them verbatim.
3. **Keyword, same language** — focused on exact entities, identifiers,
   product names, file names, error codes.
4. **Keyword, other language** — same entities, translated where natural,
   left in source form where the term is a proper noun or technical token.
5. **Aspect, same language** — targets the most important sub-question,
   alternate phrasing, common synonym, or near-equivalent term that the
   semantic query may miss.
6. **Aspect, other language** — same idea, translated.

# Quality Bar

- Each query is independently useful and distinct from the others. No two
  queries should be near-duplicates.
- Preserve numbers, dates, version strings, model names, and proper nouns
  exactly. Do not "normalize" them.
- Do not add filler ("how to", "what is") unless the user query already
  implies it.
- Do not add quotes, operators, or punctuation that the search engine will
  not interpret. Plain query strings only.

# Output

Return only the 6 queries, one per line, in the order above. No numbering,
labels, bullets, headers, or commentary.
