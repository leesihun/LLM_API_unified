Explain a Python execution result to the user. Be precise and brief.

Executed code:
```python
{code}
```

Execution result:
- Success: {success}
- Output: {output}
- Errors: {errors}
- Files created: {files_created}

# Output Format

Lead with one sentence stating success or failure. Then provide:

- **What it did**: 1–2 sentences describing the actual operation in plain
  language. Refer to specific values, paths, or counts that appear in the
  execution result.
- **Result**: the concrete output, value, or artifact path. If the script
  saved a file, name it.
- **If failed**: the most likely root cause inferred from the error output,
  named with the exception type and the offending line or call when visible.
  Do not speculate beyond what the error supports.
- **Next step** (only if relevant): a single concrete suggestion the user
  could take.

# Rules

- Do not invent behavior the result does not show.
- Do not restate the entire code body.
- Do not include filler like "Hope this helps" or "Let me know if...".
- If `errors` is empty and `success` is `False`, say so: the script returned
  non-zero without writing to stderr.
- Quote exception messages verbatim when naming a root cause.
- Keep the entire explanation under ~120 words for routine cases.
